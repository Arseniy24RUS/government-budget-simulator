# Good First Issues

These tasks are intentionally small and useful for new contributors. Open an issue before starting if the change affects methodology or user-facing behavior.

## Documentation

- Add a short instructor note that explains how to compare two student strategies.
- Add a compact glossary for KPI, public support, investment horizon, and long-term situations.
- Add a README link/image existence check as an npm script.
- Improve `THIRD_PARTY_NOTICES.md` if a source or license for an asset becomes known.

## QA and Tests

- Add a test that validates README local image paths.
- Add a test that confirms `docs/methodology.md` is linked from both README language sections.
- Add a simple model-bounds test after extracting configuration from `index.html`.
- Extend i18n tests for one annual event and one long-term situation in both languages.

## UI and Accessibility

- Check keyboard focus order in the annual decision workflow.
- Improve alt text where visual assets are used.
- Review mobile layout for the budget block and policy list.
- Add accessible labels for chart regions.

## Methodology

- Propose one new annual event with budget effects, KPI effects, and a teaching rationale.
- Review one policy measure for cost/effect balance and document the reasoning.
- Draft an adaptation example for a non-Russian regional-policy course.

## Русский

Эти задачи подходят для первого вклада: они небольшие, но полезные. Если изменение влияет на методологию или поведение пользователя, сначала откройте issue.

Идеи: добавить краткую инструкцию для сравнения стратегий студентов, глоссарий KPI/поддержки/инвестиций/долгосрочных ситуаций, npm-скрипт проверки ссылок README, тесты локальных изображений, проверку ссылки на методологию в двух языковых разделах, улучшение alt-текстов и мобильной верстки, а также методологически описанное новое ежегодное событие.
